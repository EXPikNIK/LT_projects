# Architecture

## Поток выполнения
1. `main.py` загружает конфиг и создает `LoadRunner`.
2. `LoadRunner` запускает рабочие потоки и reporter.
3. Планировщик дозирует нагрузку через `RateLimiter` и кладет задания в очередь.
4. Worker берет сообщение, отправляет в Kafka через `KafkaClient`.
5. Коллбеки продьюсера инкрементят счетчики.

## Точки расширения
- `logic.py`: формирование payload и обработка доставки
- `runner.py`: алгоритм нагрузки (например, burst/poisson)
- `counters.py`: дополнительные метрики
